# Foundation PR: Epic 1 + 2 (modernize build + cleanup dead code)

## Context

Fork of CiaranMcCann/Worms-Armageddon-HTML5-Clone (Apache 2.0, 10 years dead) at /home/scott/worms. This PR lands GitHub issues [#1](https://github.com/scottmccarrison/worms/issues/1) and [#2](https://github.com/scottmccarrison/worms/issues/2) together as pure scaffolding + deletion + archival. The 74-file legacy `src/` codebase is **not** ported in this PR; it gets archived under `reference/` and will be ported module-by-module in future epics. The outcome: `npm install && npm run dev` serves a blank canvas from a clean modern stack, ready for Epic 3+ work.

## Strategy

- **Archive, don't delete, legacy code.** `src/`, `external/`, `css/` → `reference/`. Keeps original in-tree for porting lookups; excluded from build/lint via tsconfig + Biome; excluded from GitHub language stats via `.gitattributes linguist-vendored`. Once all modules are ported, a future PR removes `reference/` wholesale.
- **Apache 2.0 compliance**: `LICENSE.txt` preserved byte-identical (it's pure boilerplate; no copyright line embedded). Attribution in new `NOTICE` file + updated `README.md`.
- **Two parallel worktrees, one integration branch, one PR.** No file conflicts (verified in review).
- **Minimal CI added** (`.github/workflows/ci.yml` with typecheck+lint+build). Not full Epic 14 — just regression protection from this PR onward.

## Verification checklist (run in integration worktree)

1. `git status` → clean
2. `ls` at repo root shows: `src/ reference/ public/ data/ .github/ package.json tsconfig.json vite.config.ts biome.json .nvmrc .editorconfig .gitattributes .gitignore index.html LICENSE.txt NOTICE README.md package-lock.json node_modules/ dist/`
3. `ls reference/` → `README.md src/ external/ css/`
4. `git diff master -- LICENSE.txt` → empty
5. `npm ci` succeeds
6. `npm run typecheck` exit 0
7. `npm run lint` exit 0 (warnings OK, errors fail)
8. `npm run build` produces `dist/index.html` + hashed assets
9. `npm run dev` → `curl -s http://localhost:5173 | grep -q 'id="game"'` succeeds
10. CI workflow passes on GitHub Actions (check via `gh pr checks`)

## Risks / pre-flight checks

1. **Verify no pre-existing branches/worktrees conflict**: `git worktree list`, `gh pr list --repo scottmccarrison/worms`
2. **Confirm master is up to date with origin** before branching: `git fetch && git status -uno`
3. **Lockfile**: `npm install` on Node 20.20.2 will commit `package-lock.json`; CI uses `npm ci` for deterministic installs
4. **Biome schema URL** pins to `1.9.0` — if npm resolves a higher patch, harmless JSON-schema IDE warning only
5. **strict mode flags**: using standard `strict: true` only. Skipping `noUncheckedIndexedAccess` and `exactOptionalPropertyTypes` — too strict for incoming game code, can add later
6. **`.gitignore` removes `*.js` rule**: verified no unintended consequences; the tracked legacy .js files (Stats.js, Box2D, jQuery, SoundBufferLoader.js) stay tracked after the move to `reference/`

  throw new Error("#game canvas missing");

  throw new Error("2D canvas context unavailable");

ctx.font = "24px system-ui, sans-serif";
ctx.textAlign = "center";
ctx.textBaseline = "middle";

# Reference (archived)

In-tree archive of the original
[Worms-Armageddon-HTML5-Clone](https://github.com/CiaranMcCann/Worms-Armageddon-HTML5-Clone)
codebase by Ciaran McCann (2012-2013), preserved verbatim from the fork point.

## Why keep it

The new implementation under `/src` is being ported module-by-module
from this reference. Keeping the original here makes cross-referencing
fast: click through to `reference/src/weapons/HandGrenade.ts` instead of
digging through `git show`.

## Status

- **Not built.** `tsconfig.json` excludes this tree; Vite does not import it.
- **Not linted.** `biome.json` ignores it.
- **Not running.** The assets, HTML shell, and build system that drove this
  code are removed or replaced. It only compiles with the original MSBuild pipeline.

## When to delete

Once every module referenced here has been ported to `/src`, this directory
gets removed in a dedicated "retire-legacy" PR.
